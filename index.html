Comment choisir un bon framework JavaScript ?

Article rédigé par : Mikhail Trefilov
02 Mai 2021 Développement web

Aujourd'hui, il n'est plus possible de créer un site Web qui serait attrayant et compréhensible pour l'utilisateur, et le créer rapidement et efficacement en JavaScript pur sans utiliser frameworks différents. Néanmoins, il existe aujourd’hui plusieurs frameworks JavaScript pour développent UI et pour développent backend. Dans cet article seules les frontend frameworks JavaScript seront présenté.

Pour répondre à la question, « Comment choisir un bon framework JavaScript pour créer des sites Web? » Il est nécessaire de comprendre, d'abord, quels frameworks sont en demande dans le marché du travail des développeurs frontend, et puis comprendre quels sont les avantages et les inconvénients de ces frameworks.

Selon Google Trends, la tierce gagnant des frontend frameworks JavaScript plus populaire se compose de tous les connus React , VueJS et Angular (JQuery n’est pas inclus dans cette liste car il est déjà obsolète). Dans cet article, nous allons examiner l’usage, les avantages et les inconvénients de ces frameworks.

React

Quant à React c'est le framework le plus utilisé et le plus demandé sur le marché du travail des développeurs frontend (selon le site indeed.com). React a été crée par Facebook en 2013, Et bien évidemment il est utilisé dans Instagram, WhatsApp et Facebook

React est considère parfois comme JavaScript librairie au lieu de framework complet.

Avantages

  • React a été créé et géré par Facebook, ce qui est sans aucun doute son avantage, car en tant que développeur frontend, vous n'aurez pas à craindre de cesser de le soutenir.
  • La communauté de React est vraiment grande, c’est pourquoi vous pourriez l’apprendre facilement grâce a nombreux cours proposées par la communauté.
  • React utilise DOM virtuelle, qui permet améliorer la performance du projet.

Inconvénients

  • Le développement de nouvelles fonctionnalités React est extrêmement rapide, ce qui nous amène au premier inconvénient de ce framework, c’est une petite quantité de documentation officielle (cas même, la communauté des développeurs très nombreux de React produit des instructions eux-mêmes).
  • Une autre caractéristique controversée de React à mon avis est l'intégration de la technologie JSX , qui vous permet d'écrire du code React directement dans la mise en page HTML. Peut-être pour le développeur débutant, cette fonction sera inhabituelle.
  • React a des problèmes avec la migration aux appareils portables, car le code dont est écrit pour sites web ne convient pas aux IOS et Android. Pour ce but il existe React Native.

VueJS

VueJS est un Framework moins commun et recherché, mais cela ne signifie pas qu'il est inutile. Comme les frameworks précédents, Vue.JS est capable de créer des interfaces utilisateur. Ce framework a été créé en 2014 par Evan Yu, un ancien employé de Google.

Avantages

  • Pour chaque développeur frontend débutant et déjà expérimenté, l'un des principaux avantages de ce framework sera sa taille extrêmement petite, occupant seulement ~20-30Ko de mémoire (la taille du projet final sera inférieure à celle de React et Angular), Vue est particulièrement adapté pour créer des versions mobiles du site tout en économisant du trafic mobile.
  • Dans le même but, VueJS utilise DOM virtuelle (comme React) et ne modifie que les éléments qui doivent être modifiés sans recharger la page. Cette approche a un effet positif sur les performances de l'application et l'économie du trafic Internet.

Inconvénients

Cependant, en choisissant VueJS pour votre projet, vous devez prendre en compte, en tant que développeur, le fait qu'il s'agisse d'un Framework assez jeune et que la quantité de documentation officielle est très faible, par rapport aux frameworks mentionnés dans cet article. En outre, ce framework est popularisé en Chine, car les sites Google et Facebook eux-mêmes sont interdits (respectivement, React et Angular y ont été distribués pas très bien). C'est pourquoi la plupart de la documentation est écrite en chinois, ce qui est un inconvénient pour les développeurs qui ne parlent pas cette langue.

Angular

Angular est un Framework très populaire créé et maintenu par Google. Son but principal est de faciliter le processus de création de page web. Vous pouvez l’essayer pour création de l’application web progressive (PWA). Ensuite ce framework est convenable pour les grands projets avec la structure très compliqués, ceux qui n’ont pas besoin de la diminution de la taille du projet finale.

Avantages

  • Angular est le projet créé et utiliser par Google, donc il est vraiment fiable du point de vue de service à long terme.
  • Le nombre de fonctionnes d’Angular pur va vous choquer, il existe de nombreuses moyens d'accomplir n'importe quelle tâche.
  • Angular utilise liaison de données bidirectionnelle (two-way data binding) qui synchronise le model et la vue. Ça permet aux développeurs de n’écrire pas du code supplémentaire pour synchroniser les données et la vue d’application.

Inconvénients

  • En termes de performances et de volume du projet, Angular perd à VueJS, car sa taille finale dépend de la taille du projet lui-même, et peut prendre environ 500Ko , ce qui n’est pas très bien pour les application d’une seule page (SPA).
  • Angular est infâme pour ses problèmes avec migration entre versions. AngularJS ce n’est pas le même chose que la version d’Angular actuelle, car il y avait plusieurs changements drastiques qui nous gênent mettre nos applications à jour rapidement et facilement.

Après avoir examiner tous les trois frameworks plus populaires, nous pouvons dire que si vous voulez faire vos projets plus léger possible, VueJS est pour vous.

Si vous cherchez l’outil pour la création des grands projets ou il y a beaucoup de documentation officielle disponible, vous devez considérer Angular comme le bon framework.

React est pour les développeurs qui veulent toujours trouver des projets intéressants en freelance. Ce framework est assez facile à apprendre, donc les développeurs débutant le choisissent souvent comme l’outil principal.
